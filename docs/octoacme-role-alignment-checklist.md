# OctoAcme — Role Alignment Checklist

## Purpose
Use this checklist to confirm role coverage, ownership, and handoffs before work moves into delivery or release.

## When to use it
- At project kickoff
- During planning for a new milestone or release
- Before release readiness review

## Named Owners
- Project Manager:
- Product Manager:
- Engineering Manager / Technical Lead:
- QA / Test Engineer:
- Designer / UX Researcher:
- Support / Operations Representative:
- Stakeholder / Sponsor:

## Lifecycle Checks

### Initiation
- [ ] Problem statement and desired outcome are documented
- [ ] Product Manager and Stakeholder / Sponsor agree on success measures
- [ ] Project Manager confirms scope, timeline, and key dependencies

### Planning
- [ ] Engineering Manager / Technical Lead reviewed feasibility and technical risks
- [ ] Designer / UX Researcher reviewed the user flow or discovery needs
- [ ] QA / Test Engineer defined the initial test approach
- [ ] Support / Operations Representative identified rollout or support-readiness needs

### Execution
- [ ] Developers, QA, and Technical Lead agree on acceptance and Definition of Done
- [ ] Risks, blockers, and dependency owners are current in the project tracker
- [ ] Stakeholder / Sponsor decisions or approvals are captured when needed

### Release Readiness
- [ ] QA / Test Engineer completed planned validation or documented open risks
- [ ] Support / Operations Representative confirmed runbooks, monitoring, and support notes
- [ ] Project Manager confirmed communication timing and release ownership
- [ ] Product Manager approved customer-facing readiness and known-issue messaging

### Retrospective / Follow-up
- [ ] Post-release feedback from support, stakeholders, and the delivery team is captured
- [ ] Action items have clear owners and target dates
